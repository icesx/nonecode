#### P操作

P(S)=S-1，占据资源，P(S)小于零进入**阻塞**队列。等于0继续执行

#### V操作

V(S)=S+1,释放资源，检查释放有人排队。通知阻塞队列。释放小于等于0，小于零的绝对值是排队数量。

先发唤醒通知，继续执行V操作后续逻辑，被通知的进程进入唤醒状态

### 事物
1. read uncommitted
脏读：读取到了其他事物修改但是未提交的内容

2. read committed
    不可重复读：事物未提交，结果不可见

3. repeatable read
    同一事物的多个实例，并发读取数据时，会看到同样的行。
    导致幻读：一个事物读取范围行，另外一个事物增加行，用户再读取该范围数据行时，出现幻影行。

4. serializable
    事物排序

5. 汇总

| 隔离级别	| 脏读（Dirty Read）	| 不可重复读（NonRepeatable Read）| 幻读（Phantom Read）|
| ---- | ---- | ---- | ---- |
| 未提交读（Read uncommitted）|	可能	|可能|	可能 |
| 已提交读（Read committed）	|不可能|	可能|	可能 |
| 可重复读（Repeatable read）|	不可能|	不可能|	可能 |
| 可串行化（Serializable ）	|不可能	|不可能|	不可能 |

### 企业应用集成(EAI)（重要）

#### 切入点

> 集成点不一样

1. 表示集成：集成点在界面，没有明确的技术
2. 数据集成：避开业务逻辑，**集成点在中间件（JDBC）**，需要对数据库进行操作。或者数据仓库。*主体数据库，可以跑业务系统*
3. 控制集成：集成点在业务逻辑，API集成。一个系统调用另一个系统的接口。可以用消息。
4. 业务流程集成：集成点在业务逻辑，B2B，可以对接到企业外部。会考虑业务流程优化。

#### 消息传递方式

1. 消息集成：传递消息进行对接，异步、交互频繁
2. 共享数据库：交互频繁、同步
3. 文件传输：数据量大，不频繁。
